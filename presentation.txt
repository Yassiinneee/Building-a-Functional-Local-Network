3.2 ETHERNET FRAME STRUCTURE
-----------------------------------------------------------

Suppose PC1 wants to send a print job to the printer.

PC1 IP Address:
192.168.10.10

Printer IP Address:
192.168.10.50

The Ethernet frame contains:

+------------------------------------------------+
| Destination MAC | Source MAC | Data | FCS |
+------------------------------------------------+

Example:

Destination MAC:
00:AA:BB:CC:DD:30

Source MAC:
00:AA:BB:CC:DD:21

Data:
Print Request

FCS:
Frame Check Sequence used for error detection.

Communication Process:

Step 1:
PC1 checks whether the printer is on the same subnet.

Step 2:
PC1 sends an ARP request to discover the printer's MAC address.

Step 3:
The printer responds with its MAC address.

Step 4:
PC1 creates the Ethernet frame.

Step 5:
The switch forwards the frame to the printer.

Step 6:
The printer receives the frame and processes the print job.

3.3 COLLISION HANDLING
-----------------------------------------------------------

A) WIRED ETHERNET

Modern switched Ethernet uses Full-Duplex communication.

Characteristics:

- No shared medium
- Separate collision domain per switch port
- No collisions occur

Older Ethernet used CSMA/CD:

Carrier Sense Multiple Access with Collision Detection

Process:

1. Listen before transmitting.
2. Transmit if medium is free.
3. Detect collision.
4. Stop transmission.
5. Send jam signal.
6. Wait random time.
7. Retransmit.

B) WI-FI NETWORKS

Wi-Fi uses CSMA/CA:

Carrier Sense Multiple Access with Collision Avoidance

Process:

1. Listen to channel.
2. Wait random backoff period.
3. Transmit frame.
4. Receive acknowledgment (ACK).
5. Retransmit if ACK not received.

CSMA/CA reduces collisions because wireless devices
cannot detect collisions efficiently.

4.5 DHCP ROLE
-----------------------------------------------------------

DHCP stands for:

Dynamic Host Configuration Protocol

DHCP automatically assigns:

- IP Address
- Subnet Mask
- Default Gateway
- DNS Server

DHCP Process:

1. DHCP Discover
2. DHCP Offer
3. DHCP Request
4. DHCP Acknowledgment

Advantages:

- Automatic configuration
- Reduced human errors
- Easier administration
- Faster deployment

DNS Response:

STEP 2 - PACKET CREATION
-----------------------------------------------------------

Source IP:

Destination IP:

93.184.216.34

-----------------------------------------------------------
STEP 3 - SEND TO DEFAULT GATEWAY
-----------------------------------------------------------

The destination is outside the local network.

The laptop sends the packet to:

192.168.10.129

Packet Path:

Laptop
→ Access Point
→ Router

-----------------------------------------------------------
STEP 4 - ROUTER EXAMINES IP HEADER
-----------------------------------------------------------

The router reads:

- Source IP
- Destination IP
- Protocol
- TTL
- Checksum

Destination:

93.184.216.34

-----------------------------------------------------------

Router Routing Table

Destination Network      Next Hop
------------------------------------------
192.168.10.0/24          Local

STEP 6 - NAT TRANSLATION
-----------------------------------------------------------

Private IP:

192.168.10.140

Public IP:

41.225.100.10

NAT converts the private address into a public address.

Example NAT Table

Private IP          Public IP
--------------------------------
192.168.10.140      41.225.100.10

Without NAT, private addresses cannot access
the Internet.

↓

↓

↓

Each router:

1. Reads destination IP.
2. Searches routing table.
3. Selects next hop.
4. Forwards packet.

The web server sends a response.

Path:

Web Server
↓

↓

↓
